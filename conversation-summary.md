# ملخص المحادثة - مركزي ذكي يمكن المستخدمين م ن :

**اسم المشروع**: منصة إدارة الفعاليات والأحداث في سوريا

**الرؤية**:  
توفير منصة رقمية موحدة تسهّل على الأفراد والمؤسسات في سوريا اكتشاف الفعاليات ذات الصلة، وتنظيمها، والمشاركة فيها بفعالية ومرونة. تهدف المنصة لسد الفجوة بين منظّمي الفعاليات والجمهور المهتم عبر نظام مركزي ذكي يتيح تصفّح الفعاليات، التسجيل أو الحجز، وتلقي التحديثات والتنبيهات، مع تمكين المنظمين من إدارة فعالياتهم وزيادة الوصول لجمهورهم المستهدف.

**النطاق**:  
داخل النطاق:
- تسجيل وتسجيل دخول المستخدمين (مشارك أو منظم) عبر البريد الإلكتروني أو وسائل التواصل الاجتماعي.
- استعراض وتصفّح الفعاليات الجارية والقادمة مع تصفية وبحث ذكي.
- صفحة تفاصيل الفعالية مع وصف، جدول المحتوى، الموقع، ومعلومات المنظم.
- لوحة تحكم للمنظمين لإنشاء وتعديل وحذف الفعاليات.
- لوحة شخصية للمستخدمين لمتابعة الأحداث المفضلة وتعديل البيانات.
- لوحة تحكم إدارية لمراجعة وقبول الفعاليات وإحصائيات.
- نظام تنبيهات وإشعارات للفعاليات الجديدة والتذكير.
- دعم الهواتف المحمولة واللغة العربية.
- صفحات سياسة الخصوصية والأحكام والشروط.
- تصميم بسيط يدعم الاتصال الضعيف بالإنترنت.

خارج النطاق (حالياً):
- نظام الحجز الإلكتروني الكامل (ميزة مؤجلة).
- تكاملات مع أنظمة دفع أو بوابات دفع إلكتروني.
- ميزات متقدمة مثل التذاكر الإلكترونية أو الذكاء الاصطناعي للتوصيات.

**الميزات الرئيسية**:
1. **تسجيل وتسجيل دخول المستخدمين**: عبر البريد الإلكتروني أو وسائل التواصل الاجتماعي، مع اختيار الدور (مشارك أو منظم).
2. **واجهة الفعاليات**: استعراض قائمة الفعاليات الجارية والقادمة مع ملخص (اسم، تاريخ، وقت، مكان، تصنيف، اسم الجهة المنظمة).
3. **نظام تصفية وبحث ذكي**: تصفية الفعاليات حسب المدينة، القطاع، التاريخ، وغيرها.
4. **صفحة تفاصيل الفعالية**: وصف تفصيلي، جدول المحتوى، خريطة الموقع أو عنوان نصي، معلومات المنظم، رابط التسجيل.
5. **لوحة تحكم المنظمين**: إدارة الفعاليات (إنشاء، تعديل، حذف)، إدخال بيانات الحدث.
6. **لوحة المستخدم الشخصية**: عرض الأحداث المفضلة، تعديل معلومات الحساب.
7. **لوحة تحكم إدارية**: مراجعة وقبول/رفض الفعاليات، إحصائيات بعدد الفعاليات والأكثر زيارة.
8. **التنبيهات والإشعارات**: تذكير بالفعاليات، إشعارات داخلية عن فعاليات جديدة.
9. **دعم الهواتف المحمولة**: واجهة متجاوبة.
10. **دعم اللغة العربية**: كامل.
11. **تصميم بسيط**: يدعم الاتصال الضعيف بالإنترنت.
12. **صفحات سياسات**: سياسة الخصوصية، الأحكام والشروط.

**المستخدمون والأدوار**:
- **مشارك**: تصفح الفعاليات، التسجيل، إضافة للمفضلة، تلقي الإشعارات.
- **منظم**: إدارة الفعاليات التي ينشئها (CRUD)، تتبع الحجوزات (لاحقاً)، التواصل مع المشاركين.
- **مدير/إداري**: مراجعة وقبول/رفض الفعاليات، عرض الإحصائيات، إدارة المستخدمين عند الحاجة.

**المتطلبات التقنية**:
- **الفرونت إند**: Angular (دعم اللغة العربية، Responsive Design)
- **الباك إند**: C# (ASP.NET Core Web API)
- **قاعدة البيانات**: SQL Server (افتراضي)
- **الاستضافة**: لم تحدد (أفترض استضافة سحابية أو محلية حسب الإمكانية)
- **التوثيق**: OAuth2/Identity مع دعم تسجيل الدخول الاجتماعي (Google, Facebook)
- **التخزين**: ملفات وصور الفعاليات (على سيرفر أو خدمة تخزين سحابي)
- **خرائط**: تكامل مع Google Maps أو بديل مجاني

**التكاملات**:
- تسجيل الدخول عبر البريد الإلكتروني ووسائل التواصل الاجتماعي (Google, Facebook)
- خرائط الموقع (Google Maps أو بديل)
- البريد الإلكتروني (للتنبيهات/الإشعارات)

**القيود والافتراضات**:
- ميزة الحجز الإلكتروني الكاملة مؤجلة للمرحلة القادمة.
- لا يوجد تكامل مع بوابات دفع في النسخة الأولى.
- المنصة يجب أن تدعم اللغة العربية بشكل كامل.
- التصميم يجب أن يكون بسيطًا وفعالًا على سرعات إنترنت منخفضة.
- لا توجد متطلبات أمان عالية المستوى (مثل تشفير خاص) في هذه المرحلة، فقط أمان أساسي.
- لا توجد متطلبات زمنية أو ميزانية محددة (أفترض أن النسخة الأولى مطلوبة خلال 2-3 أشهر).
- لا توجد أنظمة حالية يجب التكامل معها.
- لا توجد بيانات حساسة جداً، لكن يجب حماية بيانات المستخدمين الأساسية (كلمات المرور، البريد).

**الخيارات المختارة**:
- **الستاك التقني**: C# (ASP.NET Core Web API) للباك إند، Angular للفرونت إند، SQL Server للبيانات.
- **التوثيق**: دعم البريد ووسائل التواصل الاجتماعي.
- **الخرائط**: Google Maps أو بديل مجاني.
- **ميزة الحجز الإلكتروني**: مؤجلة للمرحلة الثانية.

**ملاحظات إضافية**:
- يجب تجهيز المنصة لتوسعة مستقبلية (إضافة ميزات مثل الدفع الإلكتروني، التذاكر، توصيات ذكية).
- يفضّل اعتماد تصميم Material أو Bootstrap للواجهة.
- يجب توثيق جميع الواجهات البرمجية (API) وتوفير دليل استخدام مبسط.
- يجب توفير وسيلة تواصل أو دعم فني مبدئي (بريد إلكتروني أو نموذج تواصل).
